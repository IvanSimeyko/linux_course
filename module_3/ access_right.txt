Права доступа.
В линуксе может быть много пользователей в системе и каждый из них может состоять в одной или нескольких группах.
Каждая группа может содержать одного или нескольких пользователей, а может быть и вообще без пользователей.
users - узнать кто сейчас в системе
less /etc/passwd - узнать какие есть пользователи в системе
    первое слово - логин пользователя 
    третья запись - уникальный идентификатор пользователя (для реальных пользователей, которые могут залогиниться в всистему UID >= 1000). Для системных служб меньше 1000

groups <имя_пользователя> - узнать группы пользователя

ls -l <путь>  - узнать права доступа. Первые три буквы - права для владельца файла, вторые - для пользоваелей из его групп, третьи - для остальных пользователей. Отсутсиве права обозначается -
-r - Read (просмотр содержимого)
w - Write (редактирование)
x - eXecute (исполнять файл или войти в директорию)

rw- rw- r-e
110 110 101   - запись в двоичном коде
420 420 401 = 664 - запись в восьмиричном коде

Изменить право доступа:
cmod [ugoa] [+-] [rwx]
[ugoa] - добавляем права текущему юзеру(u), группам юзера (g), остальным пользователям (o) или всем (a)

Можно добавить права с помощью восьмиричного кода. Популярная команда задания всех прав:
chmod 777 file.txt

Изменение обладателя файла:
chowm new_uzer:new_group <путь>

Суперпользователь:
root, группа root

Когда обычный пользователь действует от имени root:
sudo <действие> - от substitute user and do, дословно «подменить пользователя и выполнить».
